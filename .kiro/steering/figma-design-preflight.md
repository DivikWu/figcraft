# Figma UI 创建任务 — 需求确认流程

当用户请求在 Figma 中创建 UI 设计（页面、屏幕、组件等）时，**禁止直接开始创建**。必须先完成以下流程：

## 1. 需求分析与设计提案

在调用任何 Figma 创建工具之前，先整理并向用户呈现一份设计提案，包括：

- 目标平台和屏幕尺寸（iOS 402×874 / Android 412×915 / 其他）
- 风格调性（简约、活泼、商务、科技感等）
- 配色方案（主色、辅色、背景色、文字色）
- 需要创建的屏幕列表，及每个屏幕的核心功能和关键元素
- 布局思路和交互逻辑

## 2. 等待用户确认

明确询问用户是否认可该方案，或是否需要调整。

## 3. 确认后执行

只有在用户明确确认后，才开始执行 Figma 创建操作。

## 注意

- 这个流程适用于所有从零创建 UI 的场景
- 如果用户已经在 prompt 中提供了完整的设计规格（平台、配色、屏幕列表等），可以直接整理确认，不需要重复提问
- 对于小范围修改（调整颜色、改文案等）不需要走完整流程
- 设计质量规则（auto-layout、4dp 网格、图标规范等）已包含在自动加载的 `figma-essential-rules.md` 中，无需额外加载
